Git-删除仓库中已经跟踪的文件或者目录

发表于 2018-01-27 | 分类于 Git | |

在使用git仓库时，遇到过几次开始时忘记创建 .gitiginore的情况。
如果文件或文件夹被提交过以后（一般是提交前忘记在.gitignore设置忽略），就会被git仓库跟踪。这时如果直接在.gitignore设置忽略，并不会取消跟踪。commit是会被提交，push也会被推送到远端。

解决步骤：

  1. 要在 .gitignore 中设置要忽略的文件或文件夹。

  2. 删除文件或文件夹。

    ​

  3. 取消跟踪

  4. 重新提交和推送

    ​
